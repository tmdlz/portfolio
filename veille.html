Veille Technologique

Suivi de l'actualité et des évolutions technologiques

Sujet de veille

La veille technologique consiste à surveiller les évolutions techniques et les innovations dans un domaine donné. Elle permet de rester informé des nouvelles technologies, des bonnes pratiques et des menaces émergentes. Dans le cadre de mon BTS SIO SISR, j'ai choisi de concentrer ma veille sur la conteneurisation et l'orchestration cloud, un domaine central du DevOps et de l'administration moderne des infrastructures.

🐳
Conteneurisation & Orchestration Cloud
Docker, Kubernetes, CI/CD, Infrastructure as Code — Comment les conteneurs transforment le déploiement et la gestion des services informatiques en entreprise.

Articles et découvertes

Sélection d'articles marquants classés du plus récent au plus ancien

Fév. 2026 Docker

Maîtriser la conteneurisation : le guide complet pour débuter

Article pédagogique qui explique le concept de conteneurisation : pourquoi Docker a révolutionné le déploiement d'applications, la différence entre conteneur et machine virtuelle, et comment créer son premier conteneur avec un Dockerfile simple.

📎 Source : blog.stephane-robert.info
Jan. 2026 Kubernetes

Introduction à Kubernetes : pourquoi orchestrer ses conteneurs

Quand on a des dizaines de conteneurs Docker à gérer, il faut un outil pour les orchestrer. Cet article introduit Kubernetes, ses concepts de base (pods, services, deployments) et explique pourquoi il est devenu le standard de l'industrie.

📎 Source : blog.stephane-robert.info
Déc. 2025 IaC

Les bases de l'Infrastructure as Code avec Ansible

Ansible permet d'automatiser la configuration de serveurs sans installer d'agent. Cet article explique les principes de l'Infrastructure as Code, le fonctionnement des playbooks Ansible et montre comment automatiser l'installation d'un serveur web en quelques lignes YAML.

📎 Source : blog.stephane-robert.info
Nov. 2025 Docker

DockerCon 2025 : les annonces clés

Résumé des annonces majeures de la conférence DockerCon 2025 : nouvelles fonctionnalités de Docker Desktop, intégration de l'IA dans les workflows de développement, et améliorations des builds multi-plateformes pour les architectures ARM.

📎 Source : Docker Blog
Oct. 2025 CI/CD

Premiers pas avec GitHub Actions

Guide officiel pour débuter avec GitHub Actions : créer son premier workflow CI/CD, comprendre les fichiers YAML de configuration, et automatiser les tests et le déploiement de son code à chaque push sur un dépôt GitHub.

📎 Source : GitHub Docs
Sept. 2025 Kubernetes

K3s : Kubernetes léger pour les homelabs

K3s de Rancher est une distribution Kubernetes légère qui fonctionne avec moins de 512 Mo de RAM. Idéal pour les homelabs et les Raspberry Pi, il permet d'apprendre Kubernetes sans avoir besoin d'une infrastructure coûteuse.

📎 Source : CNCF
Juil. 2025 Conteneurs

Podman vs Docker : quelle différence ?

Podman est une alternative à Docker développée par Red Hat. Contrairement à Docker, Podman fonctionne sans démon (daemon) et peut exécuter des conteneurs sans droits root, ce qui améliore la sécurité. L'article compare les deux outils et leurs cas d'usage.

📎 Source : Red Hat
Mai 2025 Cloud

Le cloud souverain en France : enjeux et acteurs

Face à la domination des géants américains (AWS, Azure, GCP), la France développe des solutions cloud souveraines. OVHcloud, Scaleway et Infomaniak proposent des hébergements conformes au RGPD. Le label SecNumCloud de l'ANSSI garantit un niveau de sécurité élevé.

📎 Source : L'Usine Digitale
Mars 2025 IaC

Terraform pour les débutants : créer son infrastructure en code

Terraform permet de décrire son infrastructure (serveurs, réseaux, bases de données) sous forme de code. Cet article explique le langage HCL, les commandes de base (init, plan, apply) et montre comment provisionner une machine virtuelle automatiquement.

📎 Source : blog.stephane-robert.info
Jan. 2025 Sécurité

Sécurité des conteneurs : les bonnes pratiques

Les conteneurs ne sont pas sécurisés par défaut. Cet article présente les bonnes pratiques essentielles : utiliser des images officielles et à jour, ne pas exécuter en root, scanner les vulnérabilités avec Trivy, et limiter les permissions avec les security contexts.

📎 Source : blog.stephane-robert.info
